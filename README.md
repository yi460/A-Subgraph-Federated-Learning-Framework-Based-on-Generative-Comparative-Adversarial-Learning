# 基于生成式对比对抗的子图联邦学习框架 (GCA-FL)

本仓库包含了论文《基于生成式对比对抗的子图联邦学习框架》的官方实现代码。该框架（GCA-FL）旨在解决子图联邦学习在严格保护数据隐私前提下，因数据非独立同分布（Non-IID）造成的客户端漂移与模型泛化性能下降问题。

##  核心机制

GCA-FL 框架通过“端云协同”的双轨机制打破 Non-IID 数据孤岛：
* **隐私隔离与原型提取 (客户端)**：客户端使用图卷积网络 (GCN) 提取局部子图特征，并仅将各类别节点的特征均值（即“子图原型”）上传至服务器，从物理层面上切断了反推原始图数据的可能性。
* **生成式分布重构 (服务器端)**：服务器端引入条件生成对抗网络 (cGAN)，通过生成器与判别器的极小极大博弈，模拟并生成填补特征空间空缺的伪原型。
* **原型级对比语义对齐 (服务器端)**：利用 InfoNCE 损失函数进行原型级对比学习，强制同类别真实原型与伪原型在特征超球面上保持对齐，同时推开异类原型间的距离，有效防止模式崩溃与语义偏移。
* **柔性更新策略**：采用动量平滑机制动态调节模型在“既有聚合经验”与“生成分布知识”之间的权重，防止模型发生灾难性遗忘。

## 仓库结构

代码库包含了在不同基准数据集上验证 GCA-FL 性能的独立执行脚本：

* `1.py`：在 **Cora** 数据集上的对比实验脚本（包含 FedAvg 与 Ours 的端到端训练及可视化分析）。
* `2.py`：在 **Citeseer** 数据集上的对比实验脚本（验证稀疏拓扑下的模型鲁棒性）。
* `3.py`：在 **PubMed** 数据集上的对比实验脚本（验证大规模图谱的稳定性）。
* `4.py`：**消融实验与超参数敏感性分析**脚本。基于 Cora 数据集执行论文 4.2.4 的消融实验（控制 GAN 与对比学习的开启/关闭）与 4.2.5 中对比权重 $\lambda$ 的敏感性分析。

##  环境依赖

运行本实验代码需要以下主要 Python 库：

```bash
torch
torch_geometric
numpy
matplotlib
seaborn
scikit-learn
```

*注：代码默认使用 `cuda` 进行硬件加速，若未检测到 GPU 则会自动回退至 `cpu` 运行环境。*

##  运行指南

每个脚本均包含了完整的数据处理、模型训练（FedAvg 基线与 GCA-FL）、评估及图表生成流程。您可以直接运行对应的脚本以复现论文结果：

```bash
# 运行 Cora 数据集主实验
python 1.py

# 运行 Citeseer 数据集主实验
python 2.py

# 运行 PubMed 数据集主实验
python 3.py

# 运行消融实验与超参数敏感性分析 (生成论文 4.2.4 与 4.2.5 章节的图表)
python 4.py
```

##  实验结果亮点

在高度 Non-IID 场景（Dirichlet 分布参数 $\alpha=0.5$）下，GCA-FL 在节点分类任务中取得了显著优于现有基线算法（如 FedAvg、FedPPD、FedTAD）的表现：
* **Cora**: 准确率达到 **85.2%** 。
* **Citeseer**: 准确率达到 **74.6%** 。
* **PubMed**: 准确率达到 **85.3%** 。

所有运行脚本均会自动输出 t-SNE 降维可视化图、损失动态图以及 SOTA 性能对比柱状图。
